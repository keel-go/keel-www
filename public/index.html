One module, one file, everything declared.

Every module implements a single interface. Routes, models, migrations, events — all in one place.

modules/greet/module.go 
type GreetModule struct{}

func (m GreetModule) Name() string        { return "greet" }
func (m GreetModule) DependsOn() []string { return nil }
func (m GreetModule) Models() []keel.Model { return nil }
func (m GreetModule) Migrations() []keel.Migration { return nil }
func (m GreetModule) Events() []keel.EventBinding { return nil }

func (m GreetModule) Routes() []keel.Route {
    return []keel.Route{
        keel.Get("/hello/{name}", m.Hello),
    }
}

func (m GreetModule) Hello(ctx *keel.Context) error {
    name := ctx.Param("name")
    return ctx.JSON(200, map[string]string{
        "message": "Hello, " + name + "!",
    })
}

Everything built in. Nothing bolted on.

Every feature ships with the framework — zero external dependencies, all powered by the Go standard library.

Router

Radix-tree HTTP routing with path parameters, grouping, and middleware scoping.

Database

Built-in connection manager and query builder. No external ORM needed.

Migrations

Schema versioning with up/down migrations and status tracking per module.

Authentication

JWT (HMAC-SHA256) and session management built right in.

Event Bus

Synchronous and asynchronous event dispatch between modules.

Validation

Structured validation engine with automatic 422 error marshaling.

WebSocket

Real-time bidirectional communication with first-class support.

CLI

Built-in commands for serving, migrating, scaffolding, and introspecting.

SSE

Server-Sent Events for streaming real-time data to clients.

File Storage

Abstraction layer for file operations across storage backends.

Structured Logging

Wrapper around Go's log/slog with request-scoped context.

Testing

TestApp() builder for integration testing with real modules.

The Module interface is the whole framework.

Every feature implements one contract. Optional extensions add capabilities without changing the core.

module.go 
// Core contract — every module implements this
type Module interface {
    Name()       string
    Routes()     []Route
    Models()     []Model
    Migrations() []Migration
    Events()     []EventBinding
    DependsOn()  []string
}

Up and running in minutes.

Install, create your first module, and serve.

1

Install

terminal 
go get github.com/keel-go/keel
2

Create a module

modules/greet/module.go 
type Module struct{}

func (m Module) Name() string { return "greet" }

func (m Module) Routes() []keel.Route {
    return []keel.Route{
        keel.Get("/hello/{name}", m.Hello),
    }
}

func (m Module) Hello(ctx *keel.Context) error {
    return ctx.JSON(200, map[string]string{
        "message": "Hello, " + ctx.Param("name") + "!",
    })
}
3

Wire it up and serve

main.go 
func main() {
    app := keel.New(
        greet.Module{},
    )

    app.Use(keel.Logger())
    app.Use(keel.Recovery())

    app.Serve(":8080")
}

Recommended project layout.

Convention over configuration — but the convention is just "put modules in a folder."

project structure 
myapp/
├── main.go                    # Lists all modules
├── .env                       # Environment variables
└── modules/
    ├── user/
    │   ├── module.go              # Routes, models, events
    │   ├── handlers.go            # HTTP handlers
    │   ├── models.go              # Data structures
    │   ├── migrations.go          # Schema
    │   └── events.go              # Event types
    ├── blog/                     # Same structure
    └── notification/             # Reactive module

Built for AI agents.

Keel is explicitly designed so that coding agents can understand and modify your application with minimal context.

  1. Read main.go to understand application scope
  2. Read target module.go to see complete feature
  3. Run go run . describe for machine-readable JSON
  4. Generate code using the identical Module interface
  5. Compile with go build ./... for instant verification
  6. Test with go test ./... using keel.TestApp()

Design constraints as features.

What Keel deliberately does not do is just as important as what it does.

No external dependencies

Only Go standard library and golang.org/x/ packages.

No reflection

All wiring is explicit Go code. What you see is what runs.

No struct tags for behavior

Convention-based field mapping using snake_case.

No global state

Everything hangs off the App struct. Testable by design.

No init() functions

Explicit lifecycle hooks: OnInit, OnReady, OnShutdown.

No runtime surprises

If it compiles, the wiring is correct. Period.
